# Music Drop-In System

The MathForge project supports an optional, drop-in music layer.
You can supply your own MP3 (or OGG) music loops. The app will automatically play them according to the screen context.

## How It Works
- Music files live in `public/audio/music/`.
- Files are completely optional.
- The app silently skips missing files (no crashes, no console spam).
- MP3 format is preferred. OGG is also supported as a fallback if present.
- Sound Mode (Quiet/On/Off) automatically controls music volume along with sound effects.
- The Crystal Math sound effects (SFX) remain separately generated by the Web Audio API payload, while music is played via HTML `<audio>`.
- Only one active background loop plays at a time.
- No advanced music stems (e.g. dynamic drum additions) are enabled yet.

## Supported Music Tracks

To enable music, place files with these exact names in `public/audio/music/`:

| Filename | When It Plays | Loop |
|----------|---------------|------|
| `main_menu_loop.mp3` | Home, Start, Play Menus | Yes |
| `normal_mode_loop.mp3` | Normal Mode Gameplay | Yes |
| `qmm_loop.mp3` | Quick Math Mode (QMM) Gameplay | Yes |
| `dark_mode_loop.mp3` | Dark Mode Gameplay | Yes |
| `hidden_mode_loop.mp3` | Hidden Mode Gameplay | Yes |
| `survival_mode_loop.mp3` | Survival Mode Gameplay | Yes |
| `lesson_builder_loop.mp3` | Lesson Builder screen | Yes |
| `session_summary_loop.mp3` | Session Summary screen | Yes |
| `level_complete_stinger.mp3` | Briefly between gameplay end and Summary | No |

Future support can include mode-specific layers and speed-tier momentum music.
